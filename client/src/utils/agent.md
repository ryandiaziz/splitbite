# Utility Standards

## Responsibilities
- Pure helper functions.
- Global constants.
- Formatting logic.

## Rules
- **Purity**: Functions must be pure (same input = same output, no side effects).
- **Naming**: camelCase for functions, SCREAMING_SNAKE_CASE for constants.
- **Testing**: Utilities are prime candidates for unit testing.
- **Organization**: Group by function (e.g., `formatters.ts`, `constants.ts`).
